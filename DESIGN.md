# DESIGN.md

本文件用于约束“UX 竞品深度分析”工具的后续设计和前端实现。当前项目是面向 B 端研究分析场景的暗色工作台，不是营销页；任何改动都应优先保证信息密度、清晰层级、稳定布局和连续操作体验。

## 1. 产品定位

### 产品气质

- 类型：B 端工作台 / 竞品研究控制台。
- 关键词：暗色、专业、克制、高对比、数据感、工具感。
- 第一屏目标：让用户直接进入“竞品发现 - 上传截图 - UX 纬度分析 - 输出报告”的完整流程。
- 视觉重心：侧边导航、模型配置入口、竞品发现模块、上传产品卡片、UX 维度模块、结构化报告和底部导出操作条。

### 设计原则

- 不做营销式 Hero，不使用大幅装饰插画、紫色渐变、彩色光球或纯装饰图形。
- 页面应像可长时间工作的分析台：暗色画布、稳定卡片、清晰按钮、可快速扫描的结构化内容。
- 荧光绿 `#b7ff5a` 是唯一主强调色，用于品牌、主按钮、选中态、核心状态和关键数据。
- 大面积荧光绿实底元素必须使用近黑文字 `#081007`，保证识别度。
- 卡片 hover 效果要轻，不应产生明显位移或强烈发光；无点击行为的展示卡片不应有 hover 交互。

- focus 环：`0 0 0 3px rgba(183,255,90,0.16)`。
- 阴影保持克制，优先使用低透明黑色阴影，不使用大面积高亮彩色投影。
- 纯色标签、状态标签和角色标签默认不使用描边。

## 3. 排版规范

### 字体

```css
font-family: "qount.io--Inter", Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif;
```

### 字重

- 页面主标题：`700`。
- 面板标题、卡片标题、按钮：`700` 到 `900`。
- 正文和说明：`400` 到 `600`。
- 数据、标签、状态、序号：`800` 到 `900`。

### 字号层级

| 层级 | 尺寸 | 使用场景 |
| --- | --- | --- |
| H1 | `clamp(30px, 3.4vw, 48px)` | 页面主标题。 |
| H2 | `20px` 左右 | 面板标题。 |
| 产品名称 | `20px` | 产品卡片名称编辑。 |
| 正文 | `14px-15px` | 描述、报告正文。 |
| 标签/元信息 | `12px-13px` | 角色标签、状态、表格说明。 |

### 排版约束

- 全局 `letter-spacing` 保持 `0`，不要使用负字距。
- 文本不得超出按钮、标签、卡片和图表区域；长文本应换行、省略或横向滚动。
- 雷达图维度标签不加粗，必须全部展示在黑色图表区域内。
- 表格表头和单元格保持同列对齐，不允许局部漂移。

## 4. 布局与间距

### 页面骨架

- `.app-shell`：桌面端使用左侧栏 + 右侧工作区两栏布局。
- `.sidebar`：桌面端固定在左侧并 sticky，移动端变成顶部横向导航。
- `.workspace`：承载顶部标题、模型配置入口、流程卡、竞品发现、产品上传、UX 维度分析、报告输出。
- 页面本身不应出现额外下拉滚动，滚动应主要发生在工作区。

### 间距系统

- 使用 4px 倍数间距：`4 / 8 / 12 / 16 / 20 / 24 / 28 / 32`。
- 大板块卡片内边距统一，默认桌面端为 `24px`，中小屏逐级压缩。
- 父子级间距保持一致：板块之间使用更大间距，卡片内部内容使用较小间距。
- 产品卡、报告卡、建议卡、列表卡的内部节奏应统一，不要局部突兀加大或缩小。

## 5. 关键组件规范

### 侧边栏

- 宽度约 `232px`。
- Logo 使用荧光绿实底，文字为近黑色 `#081007`。
- 导航项 hover/active 使用荧光绿文字和弱荧光绿背景，不使用描边边框。
- 导航内容固定为：竞品发现、上传截图、UX 维度分析、输出报告。
- 导航点击只更新当前定位，不应让用户在操作后被强制滚回页面顶部。

### 顶部模型配置入口

- 使用 `.status-panel.config-strip`。
- sticky 吸顶，对齐工作区最上方标题位置。
- 高度约 `74px`，圆角与当前配置卡一致。
- 标题左侧使用线性图标：已配置为对勾，未配置为感叹号；图标尺寸与标题文字高度接近。
- 图标不使用额外圆形或方块背景，颜色跟随主题绿。
- hover 为轻微上移和放大，不使用扫光。
- 状态标题、图标和说明文字必须垂直居中。

### 面板和大卡片

- 主要面板、报告容器、产品卡片使用暗色毛玻璃效果。
- 除“设置 API 模型”相关卡片外，大板块背景统一为当前毛玻璃体系。
- 大卡片 hover 使用鼠标位置的小型绿色径向渐变，透明度较低；卡片边缘不变绿。
- 无点击行为的流程展示卡片不使用 hover 交互。

### 输入框和下拉菜单

- 输入框、文本域、原生下拉和自定义下拉触发器默认去掉填充，仅保留低对比白色边框。
- 下拉触发器文案和箭头颜色与常规输入框一致。
- 下拉菜单展开面板必须有深色背景填充、阴影和足够高的 z-index，避免内容穿帮。
- 下拉菜单鼠标移出后隐藏，但 hover 热区必须覆盖触发器和菜单之间的路径，确保容易选择其他选项。
- 选中项文字使用主题色，hover 样式在目标市场和模型服务下拉中保持一致。
- 展开选填项按钮去掉填充，字号 `12px`，带箭头表达可展开状态。

### 产品上传卡片

- 每个产品最多 9 张图片。
- 上传入口和已上传缩略图保持稳定尺寸，PC 端最多单行 3 个，其他视口自适应。
- 产品名默认只展示文案和编辑图标；hover 或 focus 时展示输入边框，输入后回到文案态。
- 产品名默认文案与角色标签左对齐。
- 编辑图标使用白色，始终垂直居中。
- 图片序号使用带透明度的白色，不应抢视觉焦点。
- 已上传后的移除图标默认透明度 `0.3`，hover 后恢复不透明。
- “移除”和“抓取”按钮默认去掉填充，使用一致的弱化样式和字号。
- 产品角色标签不使用描边；本品使用主题色，竞品分别使用不同临近色相。

### 竞品发现模块

- 位于上传截图之前。
- 输入区包含本品名称、本品链接、行业关键词、目标市场、行业资料链接和行业资料文本。
- “发现竞品”是次级动作，“导入为竞品”是主动作。
- 导入竞品只填充竞品名称和链接，不覆盖本品。
- 发现竞品 loading 应出现在原“暂无候选竞品”区域，尺寸保持一致。
- 竞品卡标题、平台、类别、来源、推荐理由等长文本超出时打点，避免撑破卡片。

### UX 纬度模块

- 标题为“03. UX 纬度分析”。
- 只保留维度勾选和“生成对比报告”按钮。
- 勾选项默认不使用填充；选中态沿用浏览器 checkbox + 荧光绿 accent 的填充样式。
- 生成报告按钮位于维度下方，不应与模型配置或上传动作混淆。

### 报告区

- 报告结构保持：排名、矩阵、整体最优结论、雷达图、交互细节、产品优劣势、行动建议、资料来源。
- 横向对比矩阵不展示分数，只展示结论内容。
- 整体最优结论区域使用指定暗色渐变背景，不拆成两个正文区域。
- 正文总结按产品分序号展示。
- 产品标注颜色必须与上传板块一一对应：本品用主题色，竞品使用对应竞品色，不用灰色区分。
- 行动建议正文按序列表述；优先级标签与卡片内容在垂直方向居中。
- 行动建议优先级：低用品牌色，中用黄色，高用红色。

### 雷达图

- 雷达图区域宽度保持稳定，高度根据右侧生成内容自适应。
- 雷达图不展示额外缩放按钮，应根据外框区域自适应缩放。
- 每一维度信息都必须完整展示，不能超出黑色图表区域。
- 维度文字不加粗。
- 图例颜色与上传板块产品标签一一对应。

### 底部导出操作条

- 导出按钮默认不展示；只有生成报告后才展示。
- 生成报告后显示固定在底部的通栏矩形背景。
- 通栏高度与模型配置卡一致，约 `74px`。
- 通栏背景为 `rgb(31, 31, 31)`。
- 导出按钮放在右下角，与配置卡/工作区右边缘对齐。
- “导出 HTML”为次级按钮，透明背景，仅保留边框和文字；hover、focus、active 也不出现填充。
- “导出为 PDF”为主按钮，荧光绿实底，近黑文字。
- 按钮尺寸、圆角和“更改配置”按钮保持同一体系。

## 6. 交互规范

### 通用反馈

- hover：可点击卡片使用低透明度绿色径向渐变或轻微提亮。
- focus：使用荧光绿边框和弱 focus 环。
- disabled：降低透明度，保留布局尺寸。
- loading：按钮文案变化，避免重复提交。

### 滚动与定位

- 点击上传图片、导入竞品、生成报告、导出按钮等操作后，不应自动锚定回竞品发现或页面顶部。
- `render()` 更新 UI 时必须尽量保留 `.workspace` 滚动容器和 scrollTop。
- toast 或状态提示不应触发页面回滚。
- 导航定位符点击才切换 active 状态，不应固定停留在第一个板块。

### 上传与预览

- 点击上传入口选择图片。
- 上传后缩略图向右排列，超出后折行。
- 点击缩略图打开预览。
- 点击预览关闭按钮关闭；点击图片本体不关闭弹层。
- 图片文件名在底部居中，过长时省略。

### 模型配置

- 未配置模型时，配置入口提示报告生成受限。
- 点击配置入口打开弹层。
- 弹层右上角必须有居中的关闭按钮。
- 模型服务下拉样式与目标市场下拉一致。
- 保存配置后关闭弹层，并更新配置入口文案。
- 测试连接应尽量提前暴露模型不可用、视觉能力不可用、超时、余额不足等问题。

### 报告生成

- 未配置模型时，生成按钮禁用。
- 生成中展示进度提示。
- 失败时显示可读错误，不让页面崩溃。
- 成功后展示报告，并显示底部导出操作条。

## 7. 响应式规则

### 桌面端

- `>1080px`：左侧栏 + 右侧工作区。
- 左侧栏 sticky，高度 `100vh`。
- 产品列表可多列展示。
- 上传图片区 PC 端最多单行 3 个。
- 表格保持列宽，使用横向滚动承载宽内容。

### 中等屏幕

- `<=1080px`：布局变为单列。
- 侧边栏变成顶部横向区域。
- 导航横向滚动，避免换行挤压。
- 产品卡片、报告卡片、分析卡片统一单列。

### 移动端

- `<=760px`：页面边距压缩，工作区 padding 使用更小间距。
- H1 降为约 `26px`。
- 面板圆角降为 `16px`，padding 降为 `16px`。
- 配置入口改为纵向结构，按钮满宽。
- 输入框与按钮在窄屏下改为单列。
- 上传图片区使用 `repeat(auto-fit, minmax(104px, 1fr))`。
- 表格和报告矩阵必须允许横向滚动，不强行压缩文字。
- 图片预览弹层必须适配小屏，关闭按钮保持在可点击区域内。
- 底部导出操作条保持固定，按钮在右侧可触达区域内，不遮挡主要操作。

## 8. 打印与 PDF

- 打印时隐藏侧边栏、模型配置入口、输入面板、进度面板和底部导出操作条。
- PDF 保持暗色报告风格。
- 报告卡片和表格避免跨页断裂。
- 不要在 PDF 中展示“更改模型/API”相关功能。

## 9. 不应破坏的约束

- 不要把首页改成营销页，首屏必须是可用工作台。
- 不要改变主色 `#b7ff5a`。
- 不要让荧光绿实底按钮、Logo 或纯色标签使用白色文字。
- 不要移除暗色 B 端工作台风格。
- 不要移除前置“竞品发现”模块，除非用户明确要求回退。
- 不要新增额外付费搜索 API Key；默认只用 App Store 公开检索、用户资料和现有模型配置。
- 不要让侧边导航和模型配置入口失去 sticky/吸顶行为。
- 不要让上传入口、图片缩略图、产品链接区域发生上下错位。
- 不要让产品名称输入框 hover/focus 时跳动。
- 不要让预览弹层关闭图标依赖字体基线居中。
- 不要让表格首列、表头、单元格出现不同对齐规则。
- 不要在无报告时展示导出按钮。
- 不要恢复“功能画像生成 / 一键生成画像”的旧独立模块。
- 不要在横向对比矩阵里展示分数。
- 不要把图片上限改回 4 张；当前上限为 9 张。
- 不要用大面积紫色、青色、米色、橙棕色替换当前荧光绿暗色体系。
- 不要通过大阴影、渐变光球、装饰图形制造层级；层级应来自卡片、边框、间距和内容结构。
