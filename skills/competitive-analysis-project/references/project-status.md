# UX 竞品深度分析工具项目状态

## 1. 项目目标

把一个静态作品集改造成可实际使用的“UX 竞品深度分析”工作台。用户可以录入本品和竞品，上传产品截图或产品图片，填写/抓取产品链接，选择统一 UX 分析维度，然后生成结构化竞品分析报告。

项目目标不是营销页，而是桌面优先、兼容移动端的 B 端分析工具。核心体验是：上传截图、选择 UX 维度、输出报告。

## 3. 已完成功能

- 工作台式首页，直接进入工具界面。
- 左侧导航：上传截图、UX 纬度分析、输出报告。
- 顶部模型配置入口，支持 API Key、Base URL、模型名称配置。
- 配置弹层支持关闭、保存、重置、测试连接。
- 本品 + 最多 3 个竞品。
- 每个产品最多 9 张 JPG/PNG/WebP 图片。
- 图片上传后自适应排列，PC 端最多单行 3 个。
- 图片缩略图支持放大预览。
- 图片预览弹层关闭按钮已改为 CSS 伪元素居中。
- 产品名称支持 hover/focus 编辑，已修复点击后错位问题。
- 产品链接抓取：
  - 抓取网页正文。
  - 抓取页面截图并填充到上传区域。
  - 可从页面标题推断产品名称。
- UX 纬度分析模块：
  - 已去掉旧的独立“功能画像生成 / 一键生成画像”模块。
  - 只保留统一 UX 维度勾选和生成对比报告按钮。
- 报告生成：
  - 综合排名与 AI 分数。
  - 横向对比矩阵，不展示分数。
  - 整体最优结论。
  - 蛛网雷达图展示各产品各维度优缺点。
  - 交互细节拆解。
  - 各产品优势、劣势、场景适配。
  - 行动建议。
- 导出能力：
  - 报告生成前不展示导出按钮。
  - 报告生成后展示 HTML / PDF 导出。
  - PDF 使用暗色报告风格，不带模型配置入口。
- 已生成当前项目专属 `DESIGN.md`，并提交到 GitHub。

## 5. 重要约束

- 所有沟通默认中文。
- 不要把项目改成营销页。
- 不要改变当前功能结构：上传截图、UX 纬度分析、输出报告。
- 不要恢复旧的独立画像生成模块。
- 主色必须保持 `#00BCBC`。
- 主按钮和 Logo 文案必须是白色。
- 整体保持暗色 B 端工作台风格。
- 左侧导航和模型配置入口需要保持 sticky/吸顶行为。
- 上传入口、图片缩略图、产品链接区域不能错位。
- 产品名称 hover/focus 不能跳动。
- 预览弹层关闭图标不能依赖文本 `×` 的字体基线居中。
- 表格首列、表头、单元格必须保持统一对齐和竖线分割。
- 无报告时不展示导出按钮。
- 横向对比矩阵不展示分数。
- 图片上限为 9 张，不能改回 4 张。
- UI 修改前必须先读 `DESIGN.md`。
- 工作区可能存在用户未提交改动，编辑前必须看 `git status --short`。

## 7. 必须测试的场景

### 功能

- 本地运行 `npm run dev` 后，访问 `http://localhost:5173/`。
- `GET /api/health` 返回 `ok: true`。
- 无模型配置时，生成报告按钮禁用或提示受限。
- 点击模型配置入口可打开弹层。
- 弹层保存、关闭、重置、测试连接可用。
- 添加竞品最多到 3 个。
- 移除竞品后，本品身份仍存在。
- 上传 1 张、3 张、9 张图片。
- 删除图片后缩略图和预览对应关系正确。
- 点击产品链接抓取，能抓取正文或截图；失败时有可读错误。
- 生成报告成功后才展示导出按钮。
- 导出 HTML 和 PDF 可读。

### UI

- 产品名称默认显示 `请输入产品名称`。
- hover/focus 产品名称时，文字、输入框、编辑图标不跳动。
- 图片预览关闭按钮视觉居中。
- 配置弹层关闭按钮视觉居中。
- 上传入口与图片缩略图尺寸一致。
- 产品名称、标签、上传入口顶对齐，产品链接底对齐。
- 添加竞品按钮高度与产品卡片视觉一致。
- 表格首列居中，表头和单元格同列居中。
- 表格列间有竖线分割，表头背景不出错。
- 主按钮、Logo 文案为白色。

### 数据

- 产品名称为空、很长、中文、英文、数字、特殊字符均不破坏布局。
- 图片文件名过长时不撑破缩略图和预览底部标题。
- 多个产品、多张图片时，预览打开的是当前点击图片。
- 模型返回非法 JSON 时，页面显示清晰错误而不是崩溃。
- 兼容 Base URL：
  - OpenAI: `https://api.openai.com/v1`
  - BigModel 应使用 `https://open.bigmodel.cn/api/paas/v4`，不要把 `/chat/completions` 放进 Base URL。

### 兼容性

- 桌面宽屏。
- 桌面窄屏。
- `<=1080px` 单列布局。
- `<=760px` 移动端布局。
- 移动端打开图片预览，关闭按钮仍在可点击区域内。
- 移动端表格横向滚动可用。
- Chrome 和 Safari 至少各测一次图片预览、dialog 弹层和打印。
